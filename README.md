# 🛰️ Broadcom BCM4331 Forensic Controller (Unified v39.8)

**Hardened, forensic-grade, self-healing Wi-Fi recovery suite** for BCM4331 [14e4:4331] on Fedora 43+ (X11/GNOME).

## 📜 Forensic Evolution: How We Got Here

This project is the result of a rigorous iterative process to overcome the "recalcitrance" of standard AI-generated solutions. Previous versions (v6.1 through v39.7) suffered from systemic failures that other users and LLMs frequently struggled to resolve.

### 1. The "Reload Storm" (v39.7)
In v39.7, the dashboard was rendered unusable by an infinite reload loop. 
> **Verbatim Evidence:** `[vite] (client) page reload recovery_state.db-shm (x142)` [Source: [Session Logs](/.gemini/antigravity/brain/5b341649-f478-4269-ae54-c3dbde8a8ead/.system_generated/logs/overview.txt)]
*   **The Struggle:** Vite's default file watcher detected every SQLite write (every 10s) as a source change, triggering a browser refresh.
*   **The Fix:** Implementation of `watch.ignored` in [vite.config.ts](/vite.config.ts).

### 2. The "Sudo Wall"
Forensic commands like `tcpdump` and `modprobe` were historically blocked.
> **Verbatim Evidence:** `tcpdump: sudo: a password is required` [Source: [Grok Session Summary](/.gemini/antigravity/brain/5b341649-f478-4269-ae54-c3dbde8a8ead/.system_generated/logs/overview.txt)]
*   **The Struggle:** LLMs often "refused" or omitted the complex sudoers configuration required for non-interactive recovery.
*   **The Fix:** A hardened `/etc/sudoers.d/broadcom-control` drop-in generated by [setup-system.sh](/setup-system.sh).

### 3. The "Unchecked Networking" Recovery Failure (v39.8)
A critical bug where the system failed to recover if "Enable Networking" was unchecked in the OS applet.
> **Verbatim Evidence:** `nmcli networking: disabled` [Source: [User Feedback](/.gemini/antigravity/brain/5b341649-f478-4269-ae54-c3dbde8a8ead/.system_generated/logs/overview.txt)]
*   **The Struggle:** Standard recovery scripts assumed global networking was `on`. If manually disabled, `nmcli device connect` would fail silently.
*   **The Fix:** Explicitly forcing `nmcli networking on` during the recovery handshake in [fix-wifi.sh:L336](/fix-wifi.sh).

### 4. The "Mutex Deadlock" (v39.8)
The "Nuclear Recovery" button would fail if the background monitor was already active.
> **Verbatim Evidence:** `⚠️ Another recovery process is active (PID 1234). Waiting...` [Source: [Server Logs](/server.ts)]
*   **The Struggle:** The monitor held the hardware lock (`flock`), preventing manual overrides.
*   **The Fix:** Implementing a `FORCE` mode in `acquire_mutex` that terminates existing monitors before proceeding. [[fix-wifi.sh:L146](/fix-wifi.sh)]

## 🛠️ How It Works: Component Breakdown

### 📡 Forensic Engine (`fix-wifi.sh`)
The core recovery logic. It implements **17 Audit Points** to ensure total transparency.
*   **BKW Logic:** Detects and saves the "Best Known Working" interface to the DB.
*   **System Health:** Calculates a 0-100 health score based on interface state, pings, and DNS.
*   **Verbatim Quote:** `"Assume total system failure. Log every internal decision."` [[fix-wifi.sh:L7](/fix-wifi.sh)]
*   **Citation:** Uses `PRAGMA journal_mode=WAL;` to allow concurrent reads from the dashboard while the engine writes milestones. [[fix-wifi.sh:L123](/fix-wifi.sh)]

### 🗄️ Telemetry Hub (`server.ts`)
A high-performance Express server that bridges hardware data to the React UI.
*   **SSE Delta Streaming:** Instead of sending the full log, it tracks a `clientOffset` to stream only new lines.
*   **Verbatim Quote:** `"SSE now tracks a per-client byte offset and sends only new log content since the last tick"` [[server.ts:L227](/server.ts)]
*   **Modern ASCII Dashboard:** Overhauled terminal layout with box-drawing characters and bold telemetry. [[server.ts:L62](/server.ts)]

### 🛡️ System Integrator (`setup-system.sh`)
The "Hardening" layer that prepares the Fedora environment.
*   **Sudoers Hardening:** Grants `NOPASSWD` for the exact binary paths used in forensics.
*   **Verbatim Quote:** `"System integration complete – fully hardened"` [[setup-system.sh:L39](/setup-system.sh)]

### PID Tuning Interface
Optimize the recovery response by adjusting the PID parameters in the **"Tuning"** tab.
*   **Kp (Proportional):** Controls the immediate response to health degradation.
*   **Ki (Integral):** Eliminates steady-state error (persistent degradation).
*   **Kd (Derivative):** Dampens oscillations and prevents "overshoot" during recovery.
*   **Verbatim Quote:** `"Real-time adjustment of Kp, Ki, and Kd values via the dashboard to optimize engine response."` [[App.tsx:L450](/src/App.tsx)]
